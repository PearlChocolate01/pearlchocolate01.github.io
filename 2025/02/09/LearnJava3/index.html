04747 Java语言程序设计——选择题

发表于2025-02-09|更新于2025-02-12|计算机
|字数总计:2.1k|阅读量:

2024-10真题

  1. Java中的换行符是[\n]。
    换行符（\n）：用于将光标移动到下一行的开头，是现代操作系统中最常用的换行方式。
    回车符（\r）：用于将光标返回到当前行的开头，通常与换行符一起使用（如\r\n），以确保在某些系统（如Windows）中正确地换行。在某些特殊场景下，\r也可以用于覆盖当前行的内容。

  2. Java中的switch语法：Java的switch语句继承自C和C++，最初设计时仅支持整数类型（byte、short、char、int）和从Java 7开始支持的String类型。布尔类型（boolean）在Java中只有两个值（true和false），这种情况下使用if-else语句通常更加直观和清晰。
    Java的设计者认为，布尔类型的分支选择更适合用if-else语句来实现，而不是switch语句。switch语句更适合处理多个离散值的情况。（在C#中，switch表达式确实支持bool类型）

  3. 在Java中，声明和初始化二维数组时，int arr[][] = new int[3][2]; 这行代码定义了一个二维数组。这里的 3 和 2 分别表示二维数组的行数和列数。length表示行数。
    （C#中的写法：int[,] arr = new int[2,3];）

  4. Java中以字符方式传输的输入流对象：
    （1）FileReader：用于从文件中读取字符数据。
    （2）CharArrayReader：将字符数组转换为字符输入流，从中读取字符。
    （3）StringReader：将字符串转换为字符输入流，从中读取字符。
    （4）BufferedReader：为其他字符输入流提供读缓冲区，可以提高读取效率。（考试考这个）
    FileInputStream类和FileReader类的区别：
    （1）FileInputStream：用于读取文件的字节内容，适用于处理二进制文件。（如图片、音频文件等）
    （2）FileReader：用于读取文件的字符内容，适用于处理文本文件，并且可以与BufferedReader等流结合使用以提高读取效率。（如.txt、.java等）

  5. 在Java的Swing库中，面板（JPanel）是一个非常重要的组件，用于组织和布局其他组件（如按钮、标签、文本框等）。
    面板（JPanel）本身并不是一个布局管理器，但它是一个容器（Container），可以使用布局管理器来管理其内部组件的布局。
    布局管理器（LayoutManager）是一个接口，用于定义组件在容器中的布局规则。
    常见的布局管理器包括：FlowLayout（组件按添加顺序水平排列），GridLayout（将容器划分为网格布局）。
    JPanel不能作为顶层容器。JPanel是一个轻量级组件，通常需要被添加到一个顶层容器中，才能在屏幕上显示。Java Swing中定义了以下几种顶层容器：JFrame，JDialog，JApplet，JWindow。
    JPanel不能独立存在。
    JPanel可以嵌套。在Java Swing中，你可以将一个JPanel添加到另一个JPanel中。

  6. Java中，线程对象（Thread类）提供了许多方法来控制线程的行为。
    生命周期相关方法：
    （1）start() 启动线程
    （2）run() 线程的执行体
    （3）join() 等待线程终止
    （4）interrupt() 中断线程
    ……
    线程调度相关方法：
    （1）yield() 让出CPU时间片
    （2）sleep(long millis) 使线程暂停指定的时间
    ……
    线程同步相关方法：
    （1）notify() 唤醒一个正在等待该对象锁的线程；调用 notify() 之前，线程可能处于阻塞状态（Blocked State），等待锁的释放；调用 notify() 之后，被唤醒的线程进入就绪状态，等待调度器分配CPU时间。
    （2）wait() 调用后线程进入等待状态

    根据阻塞原因不同，阻塞分为三种：
    等待阻塞：运行状态中的线程执行wait方法，进入等待队列，等待阻塞；Java虚拟机就会把线程放到这个对象的等待池中；
    同步阻塞：线程获取同步锁失败（因为锁被其他线程占用），Java虚拟机就会把这个线程放到这个对象的锁池中；
    其他阻塞：通过调用sleep方法或者join方法或者发出I/O请求时，线程会进入阻塞状态，当sleep()状态超时，或者join()等待线程终止或者超时，或者I/O处理完毕，线程重新转入就绪状态。
    线程是操作系统能够进行运算调度的最小单位，是进程内部的一个控制序列或执行流。一个进程至少包含一个线程，称为主线程。一个进程至少包含一个线程，称为主线程，但线程本身不拥有系统资源。
    线程的主要特点包括：
    （1）线程是CPU调度的基本单元，负责执行程序中的具体任务。
    （2）线程之间可以并发执行，提高程序的效率和响应速度。
    （3）线程的上下文切换比进程更快，因为线程共享进程的地址空间。

  7. super在Java中是特殊含义关键字，表示父类

  8. 多态是面向对象编程的一个重要特性，它允许你调用一个引用变量（可以是接口类型、抽象类类型或具体类类型）所指向对象的实际类型的方法。
    在Java中，多态主要通过两种方式实现：
    （1）编译时多态（方法重载，静态）：通过方法的名称相同但参数列表不同来实现。编译器在编译时期就确定了要调用哪个方法。
    （2）运行时多态（方法覆盖，动态）：通过继承和方法覆盖（Override）实现。当子类覆盖了父类的方法时，调用该方法时实际执行的是子类的版本，而不是父类的版本。这是在运行时才确定的，因此称为动态绑定或运行时多态。

2024-04真题

  1. 初始化一维数组的正确写法
    （动态）int[] m=new int[5];
    （静态）int[] m={1,2,3,4,5};

  2. 在计算机编程中，特别是面向对象编程语言（如C++、Java等）中，“重载”（Overloading）是指允许使用相同的函数名或方法名，但通过不同的参数列表（参数的类型、数量或顺序不同）来定义多个函数或方法。这样可以在同一个作用域内实现多个功能相似但参数不同的操作。

  3. Java和C#都默认值传递（形参和实参指向同一地址），Java没有引用传递，而C#可以使用ref,out关键字实现引用传递。
    值传递，即传值调用，函数调用时传递的是参数的副本（值的拷贝，地址不同），而不是参数本身。基本数据类型传递的是本身的副本，引用类型传递的是引用地址的副本。
    引用传递，即传址调用，函数调用时传递的是参数的引用（地址）本身，而不是参数的副本。基本数据类型和引用数据类型传递的都是变量的引用（地址）。因此，对参数的修改会直接影响到原始变量。
    （只有值传递类型语言中的基本数据类型传递的是参数副本，其它都是传递的地址副本）

  4. 布局管理器中，setLayout可以设置布局管理器，setSize(int x, int y, int width, int height)设置具体的位置和大小。

  5. 静态成员变量是类的所有对象共享的数据。

  6. Swing组件和AWT组件的区别：AWT包含本地代码，Swing在实现时不包含任何本地代码

  7. 创建线程的两种方法：
    （1）定义一个继承自Thread的子类（class X extends Thread）
    （2）实现Runnable接口（class X implements Runnable）
